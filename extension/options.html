Send Safety

When enabled, MCP clients requesting skipReview: true on sendMail, replyToMessage, or forwardMessage are rejected. The client can still compose messages -- they just always open a review window before sending.

Advanced Permissions

Fine-grained policy on top of the per-tool checkboxes above. Sets per-tool constraints (force review window, max batch size, require folder argument), per-account / per-folder rules, and argument whitelists (regex / glob). Schema documented in lib/permissions.sys.mjs. Empty = no extra rules; the tool checkboxes above still apply.

Presets:

Context-savers for large inboxes: inbox_inventory returns grouped counts + sample IDs instead of raw headers (25k-message inbox → ~50 groups → <2k tokens). bulk_move_by_query combines search + move with dry_run default true -- check the count before committing. createDrafts stages up to 50 drafts at once without opening compose windows. All three are off by default; enable in Tool Access above.
